前端部署早就不只是"把 `dist/` 传上去"了。云原生平台把一整套能力做成了默认值，关键是用对。

## 边缘网络，而不是单点服务器

静态资源应该尽量靠近用户。Vercel / Cloudflare 把你的产物分发到全球边缘节点，用户从最近的节点取内容：

- 首屏延迟显著下降
- 单点故障风险消失
- 你几乎不用管运维

## 不可变部署

每次部署都是一个独立、带哈希的版本，旧版本不会被覆盖：

```bash
# 每次 push 生成一个不可变的预览部署
git push origin main
# → https://app-abc123.vercel.app  (永久可回滚)
```

这意味着**回滚只是把流量切回旧版本**，秒级、无风险。

## 缓存的两条铁律

1. **带哈希的静态资源**：`max-age=31536000, immutable`，永久缓存
2. **HTML / API**：短缓存或不缓存，保证内容新鲜

> 缓存让你快；不可变让你敢快。两者配合，才是云原生前端的底气。
